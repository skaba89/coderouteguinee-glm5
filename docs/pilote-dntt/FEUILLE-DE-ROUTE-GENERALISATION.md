# Feuille de Route Généralisation — CodeRoute Guinée (Post-Pilote 6 mois)

> **Plan de déploiement national** de CodeRoute Guinée après validation GO du pilote DNTT.
> Période : 6 mois (M0 à M6) post-décision GO.
> Cible : 16 centres agréés, ~3000 candidats/mois, couverture nationale.
> Propriétaire : Directeur DNTT + MOA + MOE.

---

## 1. Vision & Objectifs

### 1.1 Vision

À l'issue de la généralisation, **tout candidat guinéen** doit pouvoir :
- S'inscrire en ligne au permis de conduire (4 langues)
- Réserver un examen dans un centre agréé proche de chez lui
- Payer via Orange Money ou MTN MoMo
- Réviser en ligne avec cours + quiz + examens blancs
- Recevoir ses résultats électroniquement

## 4. Évolutions Infrastructure

### 4.1 Architecture cible M6

```
                        ┌─────────────────┐
                        │  Cloudflare CDN  │
                        │  (Enterprise)    │
                        └────┬─────────┬──┘
                             │         │
                ┌────────────┴─────────┴────────────┐
                │           Load Balancer            │
                └─┬───────────┬──────────────┬──────┘
                  │           │              │
            ┌─────▼──┐  ┌─────▼──┐    ┌──────▼───┐
            │Conakry │  │Kankan  │    │  Labé    │
            │  DC    │  │  DC    │    │   DC     │
            │(Active)│  │(Active)│    │ (Active) │
            └────┬───┘  └────┬───┘    └──────┬───┘
                 │           │               │
                 └───────────┴───────────────┘
                          Multi-master
                          PostgreSQL 16
                          (Citus cluster)
```

### 4.2 Composants scale-up

| Composant | Pilote (M0) | M3 | M6 |
|-----------|-------------|----|----|
| App instances | 2 | 4 | 8 |
| PostgreSQL | 1 primaire + 1 réplica | 1 primaire + 2 réplica | Cluster 3 nodes |
| Redis | 1 primaire + 1 réplica | Cluster 3 nodes | Cluster 5 nodes |
| Bande passante | 100 Mbps | 500 Mbps | 1 Gbps |
| Stockage DB | 100 GB | 500 GB | 2 TB |
- Datacenters | 2 | 2 | 3

### 4.3 Optimisations performance

- **CDN Cloudflare Enterprise** : cache images + vidéos + assets statiques
- **PostgreSQL read replicas** : répartition requêtes lecture
- **Redis cluster** : cache distribué + rate limiting
- **Compression Brotli** : réduction 30% taille réponses
- **Lazy loading images** : optimisation TTI
- **Service worker PWA** : cache intelligent offline
- **Image optimization Next.js** : WebP + responsive
- **Database connection pooling** : PgBouncer

## 11. Critères de Succès Généralisation

À M6, la généralisation est jugée réussie si :

- [ ] 16 centres opérationnels (100% cible)
- [ ] 18 000 candidats inscrits cumulés
- [ ] 3 000 candidats/mois
- [ ] Taux réussite code ≥ 60%
- [ ] Taux réussite conduite ≥ 65%
- [ ] Uptime ≥ 99.5%
- [ ] NPS ≥ 50
- [ ] 0 incident P0 non résolu < RTO
- [ ] 0 incident RGPD non notifié 72h
- [ ] Audit sécurité post-généralisation : 0 P0
- [ ] ROI positif projeté < 12 mois
- [ ] Satisfaction équipes DNTT ≥ 4/5

Si ≥ 80% critères atteints : **succès total**, préparer Phase 2 (innovation).
Si 60-80% : **succès partiel**, plan correctif 3 mois.
Si < 60% : **révision stratégie**, comité pilotage exceptionnel.
