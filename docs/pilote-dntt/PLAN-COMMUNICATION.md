# Plan de Communication — Pilote DNTT CodeRoute Guinée

> **Stratégie de communication interne et externe** du pilote DNTT.
> Couvre les 8 semaines de pilote + 2 semaines pré-lancement + 2 semaines post-bilan.
> Propriétaire : Direction DNTT + Communication DNTT.

---

## 1. Objectifs de Communication

### 1.1 Objectifs principaux

1. **Informer** le public guinéen de l'existence et des avantages de CodeRoute
2. **Recruter** les 330 candidats pilotes dans les 3 centres (Conakry-Kaloum, Kankan, Labé)
3. **Rassurer** sur la fiabilité, la sécurité et la conformité RGPD
4. **Encadrer** la communication en cas d'incident (technique ou RGPD)
5. **Préparer** la décision GO/NO-GO de généralisation (J+63)

### 1.2 Principes directeurs

- **Transparence** : communication honnête, y compris sur les difficultés
- **Proactivité** : devancer les questions plutôt que subir
- **Multilinguisme** : FR + Pular + Soussou + Malinké pour toute communication grand public
- **Inclusivité** : accessible à tous (littératie, handicap, zones rurales)
- **Mesure** : chaque communication a des KPI mesurables (portée, engagement, conversion)

## 5. Communication de Crise

### 5.1 Types de crise

| Type | Exemple | Gravité | Délai communication |
|------|---------|---------|---------------------|
| Technique | Panne plateforme > 1h | Élevée | < 30 min |
| Paiement | Webhook Orange down | Élevée | < 1h |
| Fraude | Tentative fraude examen détectée | Moyenne | < 4h (interne) |
| Sécurité | Vulnérabilité P0 audit | Critique | < 4h (interne) + externe selon impact |
| RGPD | Fuite données personnelles | Critique | < 72h (AGPD) + personnes concernées |
| Réputation | Article presse négatif | Variable | < 4h |
| Force majeure | Internet down Conakry | Critique | < 1h (SMS candidats) |

### 5.2 Cellule de crise

Composition :
- Directeur DNTT (décision finale)
- RSSI (voix sécurité)
- MOE (voix technique)
- Communication DNTT (voix externe)
- Juriste DNTT (voix légale)
- Chef projet (animation)

Activation : sur décision RSSI ou Directeur DNTT.

Lieu : salle de crise DNTT + visioconférence.

### 5.3 Procédure communication crise

1. **Détection** (< 30 min) : alerte Slack `#securite-incident` + tel RSSI
2. **Confinement** (< 1h) : cellule de crise activée, première analyse
3. **Communication interne** (< 2h) : brief équipe, consignes unifiées
4. **Communication partenaires** (< 4h) : centres pilotes, auto-écoles, AGPD si RGPD
5. **Communication candidats** (< 4h) : email + SMS (multilingue) si impact direct
6. **Communication presse** (< 8h) : communiqué validé Direction
7. **Suivi** : points hebdo jusqu'à résolution
8. **Post-mortem** (< 5 jours) : analyse, leçons, plan d'amélioration

### 5.4 Messages types

#### Indisponibilité plateforme

> "CodeRoute Guinée rencontre actuellement une indisponibilité technique. Nos équipes travaillent à la résolution. L'examen prévu à {heure} est reporté. Nous vous tiendrons informés. Pour toute urgence : {téléphone support}."

#### Incident RGPD

> "Nous vous informons qu'un incident de sécurité a affecté certaines de vos données personnelles le {date}. Conformément à la Loi L/2022/018/AN, nous avons notifié l'AGPD sous 72h. Voici les mesures que nous avons prises : {liste}. Vos droits restent garantis, contactez-nous au {email}."

#### Vulnérabilité critique

> "Dans le cadre de l'audit externe en cours, une vulnérabilité critique a été identifiée. Elle a été corrigée sous {délai}. Aucune donnée personnelle n'a été compromise. Nous restons transparents sur la sécurité de la plateforme."

| Inscriptions via réseaux sociaux | ≥ 80 candidats | UTM tracking |
| Taux ouverture emails candidats | ≥ 60% | Mail tool |
| Taux clic newsletter | ≥ 15% | Mail tool |
| NPS candidats (post-examen) | ≥ 50 | Sondage |
| Taux de satisfaction communication | ≥ 4/5 | Sondage |

### 7.2 Outils de mesure

- **Google Analytics** : trafic site, conversions
- **UTM tracking** : source inscriptions
- **Mail tool** (Sendinblue/Mailchimp) : ouverture, clics
- **Veille presse** : mention brand dans médias guinéens
- **Sondage NPS** : post-examen, post-pilote
- **Social listening** : mentions CodeRoute sur réseaux

### 7.3 Reporting

- **Hebdo** : tableau de bord communication en réunion lundi 10h
- **Mensuel** : rapport synthèse au Directeur DNTT
- **Final** : bilan communication dans rapport bilan pilote
