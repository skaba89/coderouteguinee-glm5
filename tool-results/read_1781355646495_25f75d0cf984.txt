    39‚Üí## Step 2: DESIGN - Palette & Font Plan
    40‚Üí
    41‚Üí**Before writing any ReportLab code, you MUST generate the color palette via the `palette.generate` command.** This is not optional. Do NOT hardcode hex colors. Do NOT pick colors by feel.
    42‚Üí
    43‚Üí### 2a. Generate Palette (MANDATORY)
    44‚Üí
    45‚Üí**Run this command FIRST. Copy-paste its output directly into your Python script:**
    46‚Üí
    47‚Üí```bash
    48‚Üípython3 "$PDF_SKILL_DIR/scripts/pdf.py" palette.generate --title "<document title>" --mode minimal
    49‚Üí```
    50‚Üí
    51‚ÜíThe command auto-derives the design intent from the document title, computes a mathematically harmonious palette, and outputs ready-to-paste ReportLab Python code:
    52‚Üí
    53‚Üí```python
    54‚Üí# ‚îÅ‚îÅ Color Palette (auto-generated by pdf.py palette.generate) ‚îÅ‚îÅ
    55‚Üí# Intent: neutral | Mode: minimal | Harmony: split_complementary
    56‚Üí# Contrast: text:bg=14.12 | accent:bg=3.06
    57‚Üífrom reportlab.lib import colors
    58‚ÜíACCENT       = colors.HexColor('#2f97b9')
    59‚ÜíTEXT_PRIMARY  = colors.HexColor('#252422')
    60‚ÜíTEXT_MUTED    = colors.HexColor('#8d8981')
    61‚ÜíBG_SURFACE    = colors.HexColor('#dfdad2')
    62‚ÜíBG_PAGE       = colors.HexColor('#f5f4f3')
    63‚ÜíSURFACE_RGBA  = 'rgba(0,0,0,0.03)'
    64‚Üí
    65‚Üí# ReportLab table colors
    66‚ÜíTABLE_HEADER_COLOR = ACCENT
    67‚ÜíTABLE_HEADER_TEXT  = colors.white
    68‚ÜíTABLE_ROW_EVEN     = colors.white
    69‚ÜíTABLE_ROW_ODD      = BG_SURFACE
    70‚Üí```
    71‚Üí
    72‚Üí**Options:**
    73‚Üí- `--mode minimal` (default, recommended for 50%+ of documents) | `dark` | `pastel` | `jewel` | `light`
    74‚Üí- `--harmony auto` (default, recommended) | `complementary` | `split_complementary` | `analogous` | `triadic` | `monochrome`
    75‚Üí- `--format python` (default) | `json` | `css`
    76‚Üí- `--seed <int>` - for reproducible palettes across regenerations
    77‚Üí
    78‚Üí**‚öÝÔ∏è FORBIDDEN:**
    79‚Üí- ‚ùå Writing `colors.HexColor('#xxxxxx')` with any hex value you chose yourself
    80‚Üí- ‚ùå Using `colors.red`, `colors.blue`, or any ReportLab named color for design elements
    81‚Üí- ‚ùå Skipping this step and picking colors "that look good"
    82‚Üí- ‚ùå Using different palettes for different sections of the same document
    83‚Üí
    84‚Üí**‚úÖ The ONLY acceptable way to get colors:** Run `palette.generate`, copy the output.

   109‚Üí## Step 3: EDIT - Content Transformation
   110‚Üí
   111‚Üí**Before writing any code, restructure raw text into visual roles.** This is the most impactful quality step - skipping it produces "Word document with a PDF extension".
   112‚Üí
   113‚Üí### 3a. Typographic Role Extraction
   114‚Üí
   115‚ÜíParse raw text and categorize content into visual roles:
   116‚Üí
   117‚Üí| Role | ReportLab Mapping | Description |
   118‚Üí|------|------------------|-------------|
   119‚Üí| **Hero / Display** | Cover title (36-42pt) or section opener | 1-5 words, the emotional hook |
   120‚Üí| **Kicker / Eyebrow** | Subtitle or section intro (10-12pt, muted) | Tiny context line above main heading |
   121‚Üí| **Data Sculpture** | CalloutBox or bold stat block | Extract impactful numbers (97%, $4.2M, -45ms) |
   122‚Üí| **Pull Quote** | BlockQuote (italic + left indent 24pt + accent border) | Most provocative sentence, standalone |
   123‚Üí| **Body** | Standard paragraph | Everything else |
   124‚Üí
   125‚Üí**Rule**: Before writing any body paragraph, scan for numbers with units. Every metric like "revenue grew by 12%" or "latency dropped to 45ms" MUST be extracted into a CalloutBox or metrics table - never buried in paragraph prose. See ¬ßData-to-Ink Ratio Rules below.
   126‚Üí
   127‚Üí### 3b. Section Pacing
   128‚Üí
   129‚Üí- If any H1 section has **>400 words** of continuous body text without visual breaks, split into H2 subsections or insert a visual break (table, chart, callout)
   130‚Üí- Aim for at least 1 visual element (table/chart/callout) per 2-3 pages of body text
   131‚Üí- Dense text walls are the #1 sign of poor report design

   137‚Üí## Character Safety Rule (MANDATORY)
   138‚Üí
   139‚ÜíThree rules for safe character handling in ReportLab PDFs:
   140‚Üí
   141‚Üí**a) Superscripts and subscripts**: Use `<super>`/`<sub>` tags, never raw Unicode superscript/subscript characters (e.g., `\u00b2`, `\u2082`).
   142‚Üí
   143‚Üí**b) Emoji**: ReportLab cannot render emoji. If content contains emoji, use Creative brief (HTML + Playwright).
   144‚Üí
   145‚Üí**c) Font fallback for mixed CJK/Latin text**: After registering fonts, call `install_font_fallback()` once. This automatically wraps missing-glyph characters in `<font>` tags inside every `Paragraph()`. No manual `<font name="...">` wrapping needed for mixed Chinese-English text.
   146‚Üí
   147‚ÜíMathematical/relational operators (√ó, √∑, ¬±, ‚â§, ‚àö, ‚àë, ‚âÖ, ‚à´, œÄ, ‚àÝ, Œî, etc.) are safe to use as literal characters in `Paragraph()` - both SimHei and Times New Roman have these glyphs.
   148‚Üí
   149‚Üí| Need | Correct Method | Correct Example |
   150‚Üí|------|---------------|---------|
   151‚Üí| Superscript | `<super>` tag in `Paragraph()` | `Paragraph('10<super>2</super> √ó 10<super>3</super> = 10<super>5</super>', style)` |
   152‚Üí| Subscript | `<sub>` tag in `Paragraph()` | `Paragraph('H<sub>2</sub>O', style)` |
   153‚Üí| Bold | `<b>` tag in `Paragraph()` | `Paragraph('<b>Title</b>', style)` |
   154‚Üí| Math operators | Literal char in `Paragraph()` | `Paragraph('AB ‚ä• AC, ‚àÝA = 90¬∞, and ŒîABC ‚âÖ ŒîDCF', style)` |
   155‚Üí| Scientific notation | Combined tags in `Paragraph()` | `Paragraph('1.2 √ó 10<super>8</super> kg/m<super>3</super>', style)` |
   156‚Üí
   157‚Üí```python
   158‚Üífrom reportlab.platypus import Paragraph
   159‚Üífrom reportlab.lib.styles import ParagraphStyle
   160‚Üífrom reportlab.lib.enums import TA_LEFT, TA_CENTER, TA_JUSTIFY
   161‚Üí
   162‚Üíbody_style = ParagraphStyle(
   163‚Üí    name="ENBodyStyle",
   164‚Üí    fontName="Times New Roman",
   165‚Üí    fontSize=10.5,
   166‚Üí    leading=18,
   167‚Üí    alignment=TA_JUSTIFY,
   168‚Üí)
   169‚Üí
   170‚Üí# Superscript: area unit
   171‚ÜíParagraph('Total area: 500 m<super>2</super>', body_style)
   172‚Üí
   173‚Üí# Subscript: chemical formula
   174‚ÜíParagraph('The reaction produces CO<sub>2</sub> and H<sub>2</sub>O', body_style)
   175‚Üí
   176‚Üí# Scientific notation
   177‚ÜíParagraph('Speed of light: 3.0 √ó 10<super>8</super> m/s', body_style)
   178‚Üí
   179‚Üí# Combined
   180‚ÜíParagraph('E<sub>k</sub> = mv<super>2</super>/2', body_style)
   181‚Üí
   182‚Üí# Bold heading
   183‚ÜíParagraph('<b>Chapter 1: Introduction</b>', header_style)
   184‚Üí
   185‚Üí# Math symbols
   186‚ÜíParagraph('When ‚àÝ A = 90¬∞, AB ‚ä• AC and ŒîABC ‚âÖ ŒîDEF', body_style)
   187‚Üí```
   188‚Üí
   189‚Üí**Pre-generation check - before writing ANY string, ask:**
   190‚Üí> "Does this string contain a character outside basic CJK or Mathematical/relational operators?"
   191‚Üí> If YES ‚Üí it MUST be inside a `Paragraph()` with the appropriate tag.
   192‚Üí> If it is a superscript/subscript digit in raw unicode escape form ‚Üí REPLACE with `<super>`/`<sub>` tag.
   193‚Üí
   194‚Üí**NEVER rely on post-generation scanning. Prevent at the point of writing.**
   195‚Üí
   196‚Üí**Encoding safety - before writing ANY content text:**
   197‚Üí> "Does this string contain Japanese kana („ÅÆ, „Åå, „ÅØ etc.) or rare Unicode symbols?"
   198‚Üí> If YES ‚Üí REPLACE with safe plain Chinese equivalents. Japanese kana (Hiragana U+3040-U+309F, Katakana U+30A0-U+30FF) frequently corrupt to U+FFFD (ÔøΩ) when code passes through LLM output, heredoc, or terminal encoding layers.
   199‚Üí> Common safe replacements: `Ê¥ª„ÅÆÁúüÈ≤∑`‚Üí`Ê¥ªÁºîÁúüÈ≤∑`, `ÁõêÁÉ§„ÅÆÈ≤≠È±º`‚Üí`ÁõêÁÉ§È≤≠È±º`, `ÁÉ§„ÅÆÈ∏°‰∏≤`‚Üí`ÁÇ≠ÁÉ§È∏°‰∏≤`.
   200‚Üí> If the character is genuinely needed, verify it survives a full write‚Üíread round-trip with `open(file, encoding='utf-8')`.

   263‚Üí- **In tables: ALL English content and numbers MUST use `Times New Roman`**

   276‚Üíinstall_font_fallback()

   278‚Üí# 3. Just write naturally - no manual <font> wrapping needed!

   305‚Üí## Layout & Spacing Control
   306‚Üí
   307‚Üí### Page Breaks
   308‚Üí
   309‚ÜíFollow the document type strategy defined in SOUL.md Rule 1.
   310‚Üí
   311‚Üí**Structural breaks (always - MANDATORY):**
   312‚Üí- Between cover page and TOC - **cover must NEVER share a page with anything else**
   313‚Üí- Between TOC and main content
   314‚Üí- Between main content and back cover
   315‚Üí
   316‚Üí**Content breaks (by document type):**
   317‚Üí- **Default (all document types)**: ‚ùó **Absolutely NEVER force page breaks before H1/H2** ‚Äî content flows naturally, do not insert `PageBreak()` before section headings. This is an iron rule.
   318‚Üí- Resume / contract / letter ‚Üí No content page breaks
   319‚Üí- Short article ‚Üí No content page breaks
   320‚Üí- **Exception 1**: Academic papers / textbooks / teaching plans ‚Üí `PageBreak()` before each H1 is acceptable (only if user explicitly requests academic formatting)
   321‚Üí- **Exception 2**: User explicitly requests page breaks between chapters
   322‚Üí
   323‚Üí**Anti-tear (mandatory ‚Äî but with height limit):**
   324‚Üí```python
   325‚Üífrom reportlab.platypus import KeepTogether
   326‚Üífrom reportlab.lib.pagesizes import A4
   327‚Üífrom reportlab.lib.units import inch
   328‚Üí
   329‚Üí# Maximum height for KeepTogether blocks: 40% of page height
   330‚Üí# Blocks taller than this should NOT be kept together ‚Äî they cause
   331‚Üí# the preceding page to be mostly empty when the block gets pushed down.
   332‚ÜíMAX_KEEP_HEIGHT = A4[1] * 0.4  # ~336pt ‚âà 12cm
   333‚Üí
   334‚Üídef safe_keep_together(elements):
   335‚Üí    """Wrap elements in KeepTogether only if their total height is reasonable.
   336‚Üí    If too tall, keep only the FIRST TWO elements together (e.g. title + table/image header)
   337‚Üí    to prevent title-content separation, while letting the rest paginate naturally.
   338‚Üí    """
   339‚Üí    total_h = 0
   340‚Üí    for el in elements:
   341‚Üí        w, h = el.wrap(A4[0] - 2*inch, A4[1])
   342‚Üí        total_h += h
   343‚Üí    if total_h <= MAX_KEEP_HEIGHT:
   344‚Üí        return [KeepTogether(elements)]
   345‚Üí    elif len(elements) >= 2:
   346‚Üí        # Keep at least title + first element together to prevent title orphaning
   347‚Üí        return [KeepTogether(elements[:2])] + list(elements[2:])
   348‚Üí    else:
   349‚Üí        return list(elements)
   350‚Üí
   351‚Üí# Heading stays with first paragraph
   352‚Üístory.extend(safe_keep_together([
   353‚Üí    heading_paragraph,
   354‚Üí    first_body_paragraph,
   355‚Üí]))
   356‚Üí
   357‚Üí# Image + caption: keep together ONLY if image is small
   358‚Üístory.extend(safe_keep_together([image, caption_paragraph]))
   359‚Üí
   360‚Üí# Table title + table: keep together ONLY if table is short
   361‚Üíif len(table_data) <= 15:
   362‚Üí    story.extend(safe_keep_together([table_title, table]))
   363‚Üíelse:
   364‚Üí    # Long table: just keep title with the table start
   365‚Üí    story.extend(safe_keep_together([table_title]))
   366‚Üí    story.append(table)
   367‚Üí```
   368‚Üí
   369‚Üí**‚öÝÔ∏è KEY INSIGHT:** `KeepTogether` is the #1 cause of ‚Äúempty page with chart on next page‚Äù.
   370‚ÜíWhen a `KeepTogether` block is taller than the remaining space on the current page,
   371‚ÜíReportLab pushes the **entire block** to the next page, leaving the current page mostly empty.
   372‚Üí**Always use `safe_keep_together()` to cap the maximum block height.**
   373‚Üí
   374‚Üí**‚öÝÔ∏è Note:** Do NOT use `KeepWithNext` - it is unreliable in ReportLab 4.x.

   377‚Üí* Before tables: `Spacer(1, 18)` after preceding text
   378‚Üí* After tables: `Spacer(1, 6)` before table caption
   379‚Üí* After table captions: `Spacer(1, 18)` before next content

   386‚Üí### Cover Page: HTML/Playwright Unified Cover System
   387‚Üí
   388‚Üí**‚öÝÔ∏è Report route covers are ALWAYS generated via HTML/Playwright**, using the same 7-template system defined in `typesetting/cover.md`. This ensures visual consistency across all routes (Report, Creative, Academic) and avoids the limitations of ReportLab for complex cover layouts.
   389‚Üí
   390‚Üí**Pipeline (Report route):**
   391‚Üí1. Generate **body PDF** via ReportLab (start with TOC or content - **no cover in story[]**)  
   392‚Üí2. Generate **cover HTML** following `typesetting/cover.md` 7-template system  
   393‚Üí3. **Run `poster_validate.py check-html` on cover HTML** ‚Äî fix any ERRORs before rendering (overflow:hidden, font fallback, etc.)  
   394‚Üí4. **Run `cover_validate.js` on cover HTML** ‚Äî detects text-vs-decorative-line overlaps. Non-zero exit = must fix before proceeding.
   395‚Üí   ```bash
   396‚Üí   node "$PDF_SKILL_DIR/scripts/cover_validate.js" cover.html
   397‚Üí   ```
   398‚Üí5. Render cover HTML ‚Üí single-page PDF via Playwright (`html2poster.js`) ‚Äî **NOT `html2pdf-next.js`** (which converts absolute‚Üístatic and destroys cover layout)  
   399‚Üí6. **Merge: insert cover as page 0** of body PDF using pypdf ‚Üí output single final PDF
   400‚Üí
   401‚Üí> **Why not ReportLab covers?** ReportLab is excellent for structured content (tables, paragraphs, flowables) but painful for visual design (geometric accents, precise absolute positioning, web fonts). HTML/CSS handles these natively. One cover system, one visual standard, zero inconsistency.
   402‚Üí
   403‚Üí**‚öÝÔ∏è Cover page is DEFAULT ON for all Report-route documents ‚â• 3 pages.**
   404‚Üí
   405‚ÜíThe Report route handles formal, structured documents. Readers expect a professional cover. **Always generate a cover unless the document type is explicitly excluded below.**
   406‚Üí
   407‚Üí**ALWAYS add a cover page (default behavior):**
   408‚Üí- Research reports, experiment reports, lab reports, analysis reports
   409‚Üí- White papers, industry analysis, market research
   410‚Üí- Business proposals, project plans, feasibility studies
   411‚Üí- Technical documentation, product manuals, user guides
   412‚Üí- Annual reports, financial summaries, quarterly reviews
   413‚Üí- Any formal document ‚â• 3 pages that will be shared externally or submitted
   414‚Üí
   415‚Üí**NEVER add a cover page (explicit exclusions only):**
   416‚Üí- **Resumes / CVs** - recruiters want content immediately
   417‚Üí- **Letters / memos** - single-page or short-form
   418‚Üí- **Forms / checklists / invoices** - functional documents
   419‚Üí- **Internal notes / meeting minutes** - brevity over presentation
   420‚Üí- **Documents ‚â§ 2 pages** - cover would be disproportionate
   421‚Üí- **User explicitly said "no cover"**
   422‚Üí
   423‚Üí**Rule of thumb:** If it's a report, analysis, proposal, or any formal document ‚â• 3 pages ‚Üí **add a cover**. When in doubt, add the cover. It's easier to remove a cover the user didn't want than to miss one they expected.

   468‚Üí
   469‚Üí**Insertion script (single output PDF):**
   470‚Üí```python
   471‚Üífrom pypdf import PdfReader, PdfWriter, Transformation
   472‚Üí
   473‚ÜíA4_W, A4_H = 595.28, 841.89  # A4 in points
   474‚Üí
   475‚Üídef normalize_page_to_a4(page):
   476‚Üí    """Scale a page to A4 if its dimensions don't match."""
   477‚Üí    box = page.mediabox
   478‚Üí    w, h = float(box.width), float(box.height)
   479‚Üí    if abs(w - A4_W) > 2 or abs(h - A4_H) > 2:
   480‚Üí        sx, sy = A4_W / w, A4_H / h
   481‚Üí        page.add_transformation(Transformation().scale(sx=sx, sy=sy))
   482‚Üí        page.mediabox.lower_left = (0, 0)
   483‚Üí        page.mediabox.upper_right = (A4_W, A4_H)
   484‚Üí    return page
   485‚Üí
   486‚Üídef insert_cover(cover_pdf, body_pdf, output_pdf):
   487‚Üí    """Insert cover as first page of body PDF ‚Üí single output file."""
   488‚Üí    writer = PdfWriter()
   489‚Üí    # Cover as page 1
   490‚Üí    cover_page = PdfReader(cover_pdf).pages[0]
   491‚Üí    writer.add_page(normalize_page_to_a4(cover_page))
   492‚Üí    # Body pages follow
   493‚Üí    for page in PdfReader(body_pdf).pages:
   494‚Üí        writer.add_page(normalize_page_to_a4(page))
   495‚Üí    writer.add_metadata({'/Title': 'Report Title', '/Author': 'Z.ai', '/Creator': 'Z.ai'})
   496‚Üí    with open(output_pdf, 'wb') as f:
   497‚Üí        writer.write(f)
   498‚Üí```
   499‚Üí
   500‚Üí> **‚öÝÔ∏è Size pitfall:** Playwright `page.pdf(width='794px', height='1123px')` output PDF dimensions may differ from A4 (595.28√ó841.89pt) by 1-2pt. **Do not use PIL Image.save('x.pdf')** for PNG‚ÜíPDF conversion ‚Äî DPI mapping causes severe size errors. You must call `normalize_page_to_a4()` to unify dimensions before insertion.

   515‚Üí- **Pick a layout (1-7)** from `typesetting/cover.md` that matches the document tone. No global default - every selection must be a deliberate design decision. Layout 7 for government/bidding/proposal documents.
   516‚Üí- **Maximum 4 components** on any cover. Typical recipe: Title + subtitle + 1 geometric accent + metadata.

   518‚Üí- **Background layer (optional)**: See `typesetting/cover-backgrounds.md` for 3 recipes (A=ÊûÅÁÆÄÂºßÁ∫ø, B=Â∑•Á®ãÂçÅÂ≠óËΩ¥+Á´ãÊü±, C=ÈîêËßíÂàáÂâ≤+Âá∫Ë°ÄÊñáÂ≠ó). Background renders BELOW all foreground content at 2-5% opacity.
   519‚Üí- **Mandatory `<br>` chunking**: Title MUST break every 2-4 words (CJK) or 3-5 words (English). Single-line title = FAIL.
   520‚Üí- **Bounding Box spatial dispersion**: Group elements into 2-3 bounding boxes at opposite regions (e.g., top-left + bottom-right). Never cluster everything into middle 40%.
   521‚Üí- **Safe Zone**: 12% top/bottom, 14% left/right padding on cover pages.
   522‚Üí- **No body text on cover**: Cover = title + subtitle + date + optional geometric decoration. All content goes to page 2+.

   524‚Üí### ‚öÝÔ∏è Cover Page Isolation Rule (MANDATORY)
   525‚Üí
   526‚Üí**The cover page must ALWAYS be on its own page.** Cover content and subsequent content (TOC, body text, etc.) must NEVER appear on the same page. This is a hard rule.
   527‚Üí

   533‚Üí### Alignment and Typography
   534‚Üí- **CJK body**: Use `TA_LEFT` + 2-char indent. Headings: no indent.
   535‚Üí- **Font sizes**: Body 11pt, subheadings 14pt, headings 18-20pt
   536‚Üí- **Line height**: 1.5-1.6 (leading at **1.4x font size minimum**, recommended 1.5x for CJK)
   537‚Üí  - CJK characters are taller than Latin - 1.2x causes cramped lines
   538‚Üí  - Quick reference: 10pt‚Üí14pt min/18pt rec; 14pt‚Üí20pt min/22pt rec; 36pt‚Üí50pt min/54pt rec

   541‚Üí  - Use `TA_JUSTIFY` only when ALL of:
   542‚Üí    * Text is predominantly English (‚â• 90%)
   543‚Üí    * Column width is sufficiently wide (A4 single-column body)
   544‚Üí    * Font: Western fonts (Times New Roman / Calibri)
   545‚Üí    * Chinese content: None or negligible

   694‚Üí### Table Width Management (‚öÝÔ∏è Prevent Table Overflow)
   695‚Üí
   696‚Üí**Most common table bug: columns overflow the right page margin because colWidths are hardcoded without checking available space.**
   697‚Üí
   698‚Üí**Iron Rule: The sum of all colWidths MUST NOT exceed `available_width`.**
   699‚Üí
   700‚Üí```python
   701‚Üífrom reportlab.lib.pagesizes import A4
   702‚Üífrom reportlab.lib.units import inch
   703‚Üí
   704‚Üípage_width = A4[0]  # 595.28pt
   705‚Üíleft_margin = 1.0 * inch   # 72pt (adjust to your doc's actual margins)
   706‚Üíright_margin = 1.0 * inch  # 72pt
   707‚Üíavailable_width = page_width - left_margin - right_margin  # ‚âà 451pt
   708‚Üí
   709‚Üí# ‚îÄ‚îÄ‚îÄ Method 1: Proportional widths (RECOMMENDED) ‚îÄ‚îÄ‚îÄ
   710‚Üí# Define column ratios, auto-scale to fit available_width
   711‚Üícol_ratios = [0.25, 0.40, 0.20, 0.15]  # must sum to 1.0
   712‚Üícol_widths = [r * available_width for r in col_ratios]
   713‚Üítable = Table(data, colWidths=col_widths)
   714‚Üí
   715‚Üí# ‚îÄ‚îÄ‚îÄ Method 2: Fixed + flex columns ‚îÄ‚îÄ‚îÄ
   716‚Üí# Some columns have fixed width (numbers, dates), rest fills remaining space
   717‚Üífixed_cols = {0: 80, 3: 60}  # col 0 = 80pt, col 3 = 60pt
   718‚Üífixed_total = sum(fixed_cols.values())
   719‚Üíflex_count = 4 - len(fixed_cols)  # total columns minus fixed
   720‚Üíflex_width = (available_width - fixed_total) / flex_count
   721‚Üícol_widths = []
   722‚Üífor i in range(4):
   723‚Üí    col_widths.append(fixed_cols.get(i, flex_width))
   724‚Üítable = Table(data, colWidths=col_widths)
   725‚Üí
   726‚Üí# ‚îÄ‚îÄ‚îÄ Method 3: Let ReportLab auto-calculate (simplest, less control) ‚îÄ‚îÄ‚îÄ
   727‚Üí# Pass colWidths=None - ReportLab auto-sizes based on content
   728‚Üí# ‚öÝÔ∏è Risk: may still overflow if content is too wide
   729‚Üítable = Table(data)  # no colWidths
   730‚Üí```
   731‚Üí
   732‚Üí**Checklist before adding any table:**
   733‚Üí1. Calculate `available_width` from your actual page size and margins
   734‚Üí2. Sum all colWidths - must be ‚â§ `available_width`
   735‚Üí3. For CJK text: characters are wider than Latin - budget ~12pt per CJK char at 10pt font
   736‚Üí4. If a column has long text (descriptions, policies), use Paragraph() wrapping inside cells (not plain strings) - plain strings don't wrap and will overflow
   737‚Üí5. Test with the longest expected content in each column
   738‚Üí
   739‚Üí**Common mistake - plain strings don't wrap:**
   740‚Üí```python
   741‚Üí# ‚ùå Plain string: will overflow if text is long
   742‚Üídata = [['Policy Name', 'Very long description that keeps going and going']]
   743‚Üí# ‚úÖ Paragraph: wraps within column width
   744‚Üífrom reportlab.platypus import Paragraph
   745‚Üífrom reportlab.lib.styles import getSampleStyleSheet
   746‚Üístyles = getSampleStyleSheet()
   747‚Üícell_style = styles['Normal']
   748‚Üídata = [[Paragraph('Policy Name', cell_style),
   749‚Üí         Paragraph('Very long description that keeps going and going', cell_style)]]
   750‚Üí```
   751‚Üí
   752‚Üí```
   753‚ÜíDo you need auto-TOC?
   754‚Üí‚îú‚îÄ YES ‚Üí Use TocDocTemplate + doc.multiBuild(story)
   755‚Üí‚îî‚îÄ NO ‚Üí Use SimpleDocTemplate + doc.build(story)
   756‚Üí```
   757‚Üí
   758‚Üí| Requirement | DocTemplate | Build Method |
   759‚Üí|-------------|-------------|--------------|
   760‚Üí| Multi-page with TOC | `TocDocTemplate` | `multiBuild()` |
   761‚Üí| Single-page or no TOC | `SimpleDocTemplate` | `build()` |
   762‚Üí| With Cross-References (no TOC) | `SimpleDocTemplate` | `build()` |
   763‚Üí| Both TOC + Cross-References | `TocDocTemplate` | `multiBuild()` |
   764‚Üí
   765‚Üí**‚öÝÔ∏è CRITICAL**:
   766‚Üí- `multiBuild()` is ONLY needed when using `TableOfContents`
   767‚Üí- Using `build()` with `TocDocTemplate` = TOC won't work
   768‚Üí- Using `multiBuild()` without `TocDocTemplate` = unnecessary overhead

   798‚Üí## Auto-Generated Table of Contents
   799‚Üí
   800‚Üí### ‚ùå FORBIDDEN: Manual Table of Contents
   801‚Üí**NEVER manually create TOC with hardcoded page numbers.**
   802‚Üí
   803‚Üí### ‚öÝÔ∏è MANDATORY: TOC requires a cover page
   804‚Üí**Unless the user explicitly requests no cover, if a document has a Table of Contents, it MUST have a cover page.** Structure: Cover (page 1) ‚Üí TOC (page 2) ‚Üí Content (page 3+). Do not generate a TOC without a preceding cover page.
   805‚Üí
   806‚Üí### ‚úÖ ALWAYS use auto-generated TOC:
   807‚Üí
   808‚Üí```python
   809‚Üífrom reportlab.platypus import SimpleDocTemplate, Paragraph, PageBreak, Spacer
   810‚Üífrom reportlab.platypus.tableofcontents import TableOfContents
   811‚Üífrom reportlab.lib.utils import simpleSplit
   812‚Üíimport hashlib
   813‚Üí
   814‚Üíclass TocDocTemplate(SimpleDocTemplate):
   815‚Üí    def afterFlowable(self, flowable):
   816‚Üí        if hasattr(flowable, 'bookmark_name'):
   817‚Üí            level = getattr(flowable, 'bookmark_level', 0)
   818‚Üí            text = getattr(flowable, 'bookmark_text', '')
   819‚Üí            key = getattr(flowable, 'bookmark_key', '')
   820‚Üí            # MUST pass key as 4th element - without it, TOC entries won't have clickable links
   821‚Üí            self.notify('TOCEntry', (level, text, self.page, key))
   822‚Üí
   823‚Üídoc = TocDocTemplate("document.pdf", pagesize=letter)
   824‚Üístory = []
   825‚Üí
   826‚Üítoc = TableOfContents()
   827‚Üítoc.levelStyles = [
   828‚Üí    ParagraphStyle(name='TOCHeading1', fontSize=14, leftIndent=20, fontName='Times New Roman'),
   829‚Üí    ParagraphStyle(name='TOCHeading2', fontSize=12, leftIndent=40, fontName='Times New Roman'),
   830‚Üí]
   831‚Üístory.append(Paragraph("<b>Table of Contents</b>", styles['Title']))
   832‚Üístory.append(toc)
   833‚Üístory.append(PageBreak())
   834‚Üí
   835‚Üídef add_heading(text, style, level=0):
   836‚Üí    # Generate a unique bookmark key for this heading
   837‚Üí    key = 'h_%s' % hashlib.md5(text.encode()).hexdigest()[:8]
   838‚Üí    p = Paragraph('<a name="%s"/>%s' % (key, text), style)
   839‚Üí    p.bookmark_name = text
   840‚Üí    p.bookmark_level = level
   841‚Üí    p.bookmark_text = text
   842‚Üí    p.bookmark_key = key  # This key enables clickable TOC links
   843‚Üí    return p
   844‚Üí
   845‚Üí# ‚öÝÔ∏è Orphan Heading Prevention (NOT a page break rule)
   846‚Üí# If an H1 heading would appear in the bottom 15% of the page with no room
   847‚Üí# for at least a few lines of content, push it to the next page.
   848‚Üí# This is NOT "every H1 starts a new page" ‚Äî it only triggers when the heading
   849‚Üí# would be orphaned at the very bottom.
   850‚Üíavailable_height = A4[1] - top_margin - bottom_margin
   851‚ÜíH1_ORPHAN_THRESHOLD = available_height * 0.15  # ~100pt ‚âà 3.5cm ‚Äî enough for heading + 2 lines
   852‚Üí
   853‚Üídef add_major_section(text, style):
   854‚Üí    """Add an H1-level heading with orphan prevention (NOT forced page break)."""
   855‚Üí    return [
   856‚Üí        CondPageBreak(H1_ORPHAN_THRESHOLD),  # Only break if heading would be orphaned at page bottom
   857‚Üí        add_heading(text, style, level=0),
   858‚Üí    ]
   859‚Üí
   860‚Üístory.extend(add_major_section("Chapter 1: Introduction", styles['Heading1']))
   861‚Üí# ... content ...
   862‚Üí
   863‚Üídoc.multiBuild(story)  # MUST use multiBuild for TOC
   864‚Üí```
   865‚Üí
   866‚Üí**‚öÝÔ∏è CRITICAL TOC LINK RULES:**
   867‚Üí1. `afterFlowable` MUST pass `key` as the 4th element in the notify tuple - without it, TOC entries have no clickable links
   868‚Üí2. `add_heading` MUST set `bookmark_key` AND embed `<a name="key"/>` in the Paragraph text - this creates the link destination
   869‚Üí3. The key must be unique per heading - use a hash of the heading text
   870‚Üí
   871‚Üí### TOC with Leader Dots (Copy-Paste Ready)
   872‚Üí
   873‚Üí**‚öÝÔ∏è WARNING**: This manual approach creates a visual-only TOC. It has **NO clickable links** and **NO auto-updated page numbers**. Use the auto-generated TOC above (TocDocTemplate + multiBuild) whenever possible. Only use this leader-dots approach if you need very specific visual styling that the auto TOC cannot provide - and even then, prefer the auto approach.
   874‚Üí
   875‚ÜíFor a professional TOC with leader dots connecting titles to page numbers:
   876‚Üí
   877‚Üí```python
   878‚Üífrom reportlab.lib.pagesizes import A4
   879‚Üífrom reportlab.platypus import SimpleDocTemplate, Table, TableStyle, Paragraph, PageBreak
   880‚Üífrom reportlab.lib.styles import getSampleStyleSheet, ParagraphStyle
   881‚Üífrom reportlab.lib import colors
   882‚Üífrom reportlab.lib.units import inch
   883‚Üí
   884‚Üí# Setup
   885‚Üídoc = SimpleDocTemplate("report.pdf", pagesize=A4,
   886‚Üí                       leftMargin=0.75*inch, rightMargin=0.75*inch)
   887‚Üístyles = getSampleStyleSheet()
   888‚Üístyles['Heading1'].fontName = 'Times New Roman'
   889‚Üístyles['Heading1'].textColor = colors.black
   890‚Üí
   891‚Üístory = []
   892‚Üí
   893‚Üí# Calculate dimensions
   894‚Üípage_width = A4[0]
   895‚Üíavailable_width = page_width - 1.5*inch
   896‚Üípage_num_width = 50  # Fixed width for page numbers
   897‚Üídots_column_width = available_width - 200 - page_num_width
   898‚Üíoptimal_dot_count = int(dots_column_width / 4.5)  # ~4.5pt per dot at 7pt font
   899‚Üí
   900‚Üí# Define styles
   901‚Üítoc_style = ParagraphStyle('TOCEntry', parent=styles['Normal'],
   902‚Üí                          fontName='Times New Roman', fontSize=11, leading=16)
   903‚Üídots_style = ParagraphStyle('LeaderDots', parent=styles['Normal'],
   904‚Üí                           fontName='Times New Roman', fontSize=7, leading=16)
   905‚Üí
   906‚Üí# Build TOC
   907‚Üítoc_data = [
   908‚Üí    [Paragraph('<b>Table of Contents</b>', styles['Heading1']), '', ''],
   909‚Üí    ['', '', ''],
   910‚Üí]
   911‚Üí
   912‚Üíentries = [('Section 1', '5'), ('Section 2', '10')]
   913‚Üífor title, page in entries:
   914‚Üí    toc_data.append([
   915‚Üí        Paragraph(title, toc_style),
   916‚Üí        Paragraph('.' * optimal_dot_count, dots_style),
   917‚Üí        Paragraph(page, toc_style)
   918‚Üí    ])
   919‚Üí
   920‚Üítoc_table = Table(toc_data, colWidths=[None, dots_column_width, page_num_width])
   921‚Üítoc_table.setStyle(TableStyle([
   922‚Üí    ('GRID', (0, 0), (-1, -1), 0, colors.white),
   923‚Üí    ('LINEBELOW', (0, 0), (0, 0), 1.5, colors.black),
   924‚Üí    ('ALIGN', (0, 0), (0, -1), 'LEFT'),
   925‚Üí    ('ALIGN', (1, 0), (1, -1), 'LEFT'),
   926‚Üí    ('ALIGN', (2, 0), (2, -1), 'RIGHT'),
   927‚Üí    ('VALIGN', (0, 0), (-1, -1), 'TOP'),
   928‚Üí    ('LEFTPADDING', (0, 0), (-1, -1), 0),
   929‚Üí    ('RIGHTPADDING', (0, 0), (-1, -1), 0),
   930‚Üí    ('TOPPADDING', (0, 2), (-1, -1), 3),
   931‚Üí    ('BOTTOMPADDING', (0, 2), (-1, -1), 3),
   932‚Üí    ('TEXTCOLOR', (1, 2), (1, -1), TEXT_MUTED),  # From palette --c-muted
   933‚Üí]))
   934‚Üí
   935‚Üístory.append(toc_table)
   936‚Üístory.append(PageBreak())
   937‚Üídoc.build(story)
   938‚Üí```
   939‚Üí
   940‚Üí**MANDATORY Leader Dots Rules:**
   941‚Üí
   942‚Üí| Rule | Requirement |
   943‚Üí|------|-------------|
   944‚Üí| Column widths | ‚úÖ MUST use fixed values. Percentage-based widths are **STRICTLY FORBIDDEN**. |
   945‚Üí| Dot count | ‚úÖ MUST calculate dynamically: `int(dots_column_width / 4.5)`. Hard-coded counts are **STRICTLY FORBIDDEN**. |
   946‚Üí| Page number column | ‚úÖ MUST be at least 40pt wide. |
   947‚Üí| Dot font size | ‚úÖ MUST NOT exceed 8pt. |
   948‚Üí| Dot alignment | ‚úÖ MUST be LEFT-aligned (visual flow from title). |
   949‚Üí| Padding | ‚úÖ MUST be exactly 0 between columns. |
   950‚Üí| Bold text | ‚úÖ MUST use `Paragraph('<b>Text</b>', style)`. Plain strings like `'<b>Text</b>'` are **STRICTLY FORBIDDEN**. |
   951‚Üí| Indentation | Use leading spaces for hierarchy (e.g., `"    1.1 Subsection"`). |
   952‚Üí
   953‚Üí### TOC Post-Generation Validation (MANDATORY)
   954‚Üí
   955‚ÜíAfter generating any PDF with a Table of Contents, run:
   956‚Üí
   957‚Üí```bash
   958‚Üípython3 "$PDF_SKILL_DIR/scripts/toc_validate.py" check-pdf output.pdf
   959‚Üí```
   960‚Üí
   961‚ÜíIf any errors are returned, fix the code and regenerate. Common issues:
   962‚Üí- `TOC_ALL_SAME_PAGE` ‚Üí You used `build()` instead of `multiBuild()`, so all page numbers are stuck at 1.
   963‚Üí- `TOC_NO_ENTRIES` ‚Üí Headings are missing `bookmark_name`/`bookmark_level` attributes.
   964‚Üí- `TOC_PAGES_INVALID` ‚Üí A TOC entry references a page beyond the document's total page count.

   968‚Üí## Cross-References (Figures, Tables, Bibliography)
   969‚Üí
   970‚ÜíPre-register all figures, tables, and references BEFORE using them in text.
   971‚Üí
   972‚Üí```python
   973‚Üíclass CrossReferenceDocument:
   974‚Üí    def __init__(self):
   975‚Üí        self.figures = {}
   976‚Üí        self.tables = {}
   977‚Üí        self.refs = {}
   978‚Üí        self.figure_counter = 0
   979‚Üí        self.table_counter = 0
   980‚Üí        self.ref_counter = 0
   981‚Üí
   982‚Üí    def add_figure(self, name):
   983‚Üí        if name not in self.figures:
   984‚Üí            self.figure_counter += 1
   985‚Üí            self.figures[name] = self.figure_counter
   986‚Üí        return self.figures[name]
   987‚Üí
   988‚Üí    def add_table(self, name):
   989‚Üí        if name not in self.tables:
   990‚Üí            self.table_counter += 1
   991‚Üí            self.tables[name] = self.table_counter
   992‚Üí        return self.tables[name]
   993‚Üí
   994‚Üí    def add_reference(self, name):
   995‚Üí        if name not in self.refs:
   996‚Üí            self.ref_counter += 1
   997‚Üí            self.refs[name] = self.ref_counter
   998‚Üí        return self.refs[name]
   999‚Üí```

  1003‚Üí## Image Handling
  1004‚Üí
  1005‚Üí### Diagram Embedding Rules (MANDATORY)
  1006‚Üí
  1007‚Üí**Figures are block-level Flowables.** Every image/diagram in ReportLab MUST be appended to `story` as a standalone `Image()` or wrapped in `KeepTogether([image, caption])`. Never simulate floating layouts by placing images inside Paragraph text or multi-column Table cells alongside body text.
  1008‚Üí
  1009‚Üí**Complex diagrams (>12 nodes)**: Decompose into a ReportLab Table (details) + a simplified diagram image (overview). The diagram gives the big picture; the table carries precision. See SKILL.md "Complex Diagram Strategy".
  1010‚Üí
  1011‚Üí**Diagram quality**: When generating flowcharts/diagrams for PDF embedding (via Playwright screenshot or ReportLab Drawing), follow the diagram content quality rules in SKILL.md ¬ß "Diagram Content Quality Rules". Key points: connectors must not pass through nodes, no high-saturation large fills, multi-arrow convergence must use merge pattern, font sizes must be legible at final embedded size (see context-specific minimums in SKILL.md).
  1012‚Üí
  1013‚Üí**Cross-brief diagram pipeline**: For flowcharts/architecture diagrams, generate via **Playwright+CSS** (HTML nodes + CSS layout + SVG connectors), screenshot at 2√ó device scale factor for 300dpi print quality, then embed with `Image()`. See SKILL.md ¬ß "Diagram Generation Strategy" for the full pipeline.
  1014‚Üí
  1015‚Üí**üö´ FORBIDDEN: TikZ standalone ‚Üí PNG pipeline in Report brief.** Report uses ReportLab, not LaTeX. Going through TikZ requires a LaTeX compiler, adds compilation steps, and models frequently produce broken TikZ code. Use Playwright+CSS instead - it's native to the existing cover rendering engine.
  1016‚Üí
  1017‚Üí```python
  1018‚Üí# Playwright+CSS diagram ‚Üí PNG ‚Üí ReportLab embedding
  1019‚Üí# 1. LLM generates diagram.html (CSS grid/flexbox nodes + SVG arrows)
  1020‚Üí# 2. Playwright screenshots at 2√ó scale (300dpi equivalent)
  1021‚Üí# 3. Embed:
  1022‚Üífrom reportlab.platypus import Image
  1023‚Üíimg = Image('diagram.png', width=450)  # auto height via aspect ratio
  1024‚Üístory.append(img)                       # block-level Flowable
  1025‚Üí```
  1026‚Üí
  1027‚Üí**‚öÝÔ∏è CRITICAL: Preserve aspect ratio - NEVER hardcode both width and height without reading actual image dimensions.**
  1028‚Üí
  1029‚Üí**‚öÝÔ∏è CRITICAL: Always constrain to available space - NEVER insert at original size if it exceeds container width/height.**
  1030‚Üí
  1031‚ÜíPie charts become ellipses, radar charts become diamonds, photos get stretched if you set arbitrary width/height.
  1032‚Üí
  1033‚Üí**‚Üí Full overflow prevention spec: `typesetting/overflow.md`** - read for the complete bounding box system.
  1034‚Üí
  1035‚Üí```python
  1036‚Üífrom PIL import Image as PILImage
  1037‚Üífrom reportlab.platypus import Image
  1038‚Üífrom reportlab.lib.pagesizes import A4
  1039‚Üífrom reportlab.lib.units import inch
  1040‚Üí
  1041‚Üídef embed_image(path: str, max_width: float = None, max_height: float = None) -> Image:
  1042‚Üí    """Embed image with preserved aspect ratio, constrained to container.
  1043‚Üí
  1044‚Üí    ALWAYS use this pattern. Never hardcode both width and height.
  1045‚Üí    If max_width is None, defaults to available_width (page - margins).
  1046‚Üí    """
  1047‚Üí    if max_width is None:
  1048‚Üí        max_width = A4[0] - 2.0 * inch  # Default: A4 width minus 1" margins each side
  1049‚Üí    if max_height is None:
  1050‚Üí        max_height = A4[1] * 0.35  # ~294pt ‚âà 10cm ‚Äî leaves room for caption + surrounding text on same page
  1051‚Üí
  1052‚Üí    pil_img = PILImage.open(path)
  1053‚Üí    orig_w, orig_h = pil_img.size
  1054‚Üí
  1055‚Üí    # Scale to fit within BOTH max_width and max_height
  1056‚Üí    ratio_w = max_width / orig_w if orig_w > max_width else 1.0
  1057‚Üí    ratio_h = max_height / orig_h if orig_h > max_height else 1.0
  1058‚Üí    ratio = min(ratio_w, ratio_h)
  1059‚Üí
  1060‚Üí    return Image(path, width=orig_w * ratio, height=orig_h * ratio)
  1061‚Üí
  1062‚Üí# Usage:
  1063‚Üíavailable_width = A4[0] - left_margin - right_margin
  1064‚Üíimg = embed_image('chart.png', max_width=available_width, max_height=300)
  1065‚Üístory.append(img)

  1172‚Üí## Resume / CV Template (ATS-Friendly)
  1173‚Üí
  1174‚ÜíSingle-column, clean layout optimised for Applicant Tracking Systems (ATS). No graphics, no sidebars, no colour blocks - just well-structured text that parses perfectly by HR software.
  1175‚Üí
  1176‚Üí**When to use this template:**
  1177‚Üí- Applying to corporate jobs through online portals
  1178‚Üí- Any scenario where the PDF will be machine-parsed before a human reads it
  1179‚Üí- When the recruiter explicitly asks for "a standard resume"
  1180‚Üí
  1181‚Üí**When NOT to use (use Academic brief instead):**
  1182‚Üí- Creative/design industry positions ‚Üí Academic brief (AltaCV-style)
  1183‚Üí- Academic CV with publications ‚Üí Academic brief (Academic CV)
  1184‚Üí
  1185‚Üí### Resume Design Rules
  1186‚Üí- **Target 1 page** unless user specifies otherwise
  1187‚Üí- **Margins**: `left=1.5cm, right=1.5cm, top=1.5cm, bottom=1.5cm`
  1188‚Üí- **No cover page** - content starts immediately
  1189‚Üí- **No TOC** - too short for table of contents
  1190‚Üí- **Font**: Times New Roman (English) or SimHei (Chinese)
  1191‚Üí- **Body font size**: 10-10.5pt; Name: 22-26pt; Section titles: 13-14pt
  1192‚Üí- **‚öÝÔ∏è Minimum font size: 12px (9pt) - HARD FLOOR.** No text in the entire resume may render smaller than 12px. This includes contact info, meta text, footnotes, and captions. Anything below 12px is unreadable in print and fails accessibility checks.
  1193‚Üí- **Section separator**: thin horizontal rule (`HRFlowable`)
  1194‚Üí- **Bullet style**: `‚Ä¢` with tight spacing
  1195‚Üí
  1196‚Üí### Resume Line-Break Rules (Language-Aware)
  1197‚Üí- **English**: Prefer breaking at word boundaries (spaces, hyphens). If a long word must be split to avoid excessive whitespace, break at a valid syllable boundary and insert a hyphen (`-`) - this is standard typographic practice (e.g., `experi-\nence`, `develop-\nment`). ReportLab supports `wordWrap='CJK'` only for CJK content; for English use default paragraph wrapping with `allowWidows=0, allowOrphans=0`.
  1198‚Üí- **Chinese/CJK**: Break allowed between any two CJK characters. Never break between a CJK character and its adjacent punctuation („ÄÅ„ÄÇ,)„Äã etc. must stay with the preceding character).
  1199‚Üí- **Mixed content** (e.g., "Python ÂºÄÂèëÂ∑•Á®ãÂ∏à"): Break at CJK boundaries or English word boundaries. Never split an English word in a CJK paragraph unless hyphenated.
  1200‚Üí- **Contact line**: Email, phone, location separated by `|` or `¬∑`. Each segment must stay on one line - if too long, move to next line at the separator, not mid-segment.
  1201‚Üí- **Dates and ranges**: "Jan 2022 - Present" must stay as one unit. Never break a date range across lines.
  1202‚Üí
  1203‚Üí### Resume Page-Fill Rules (Anti-Blank-Space)
  1204‚Üí- **Goal: Fill ‚â•85% of the page height.** Content should reach at least the bottom quarter of the page. A resume that stops at 60% height with blank space below = FAIL.
  1205‚Üí- **Adaptive spacing strategy** (apply in order until page is ‚â•85% filled):
  1206‚Üí  1. Increase `spaceBefore` / `spaceAfter` on section headers (from 10pt up to 18pt)
  1207‚Üí  2. Increase `leading` (line height) on body text (from 14pt up to 18pt)
  1208‚Üí  3. Increase body `fontSize` by 0.5-1pt (from 10pt up to 11.5pt max)
  1209‚Üí  4. Add a "Professional Summary" or "Key Achievements" section if none exists
  1210‚Üí  5. Increase margins slightly (from 1.5cm up to 2cm) to reduce line width and push content downward
  1211‚Üí- **Never leave a visible blank area > 3cm at the bottom of the page.**
  1212‚Üí- **If content overflows to page 2**: do the reverse - reduce spacing, tighten leading (min 12pt), reduce fontSize (min 9pt / 12px), before removing content.

  1584‚Üí## Data-to-Ink Ratio Rules (MANDATORY for Reports)
  1585‚Üí
  1586‚Üí**CRITICAL: Do NOT write long paragraphs to describe data trends.** You MUST extract metrics and trends into structured visual elements.
  1587‚Üí
  1588‚Üí### Pattern Detection Table
  1589‚Üí
  1590‚ÜíBefore writing ANY body paragraph, scan the text. If you find any of these patterns, extract them:
  1591‚Üí
  1592‚Üí| Raw text pattern | ‚ùå FORBIDDEN | ‚úÖ REQUIRED visual form |
  1593‚Üí|-----------------|-------------|----------------------|
  1594‚Üí| "revenue grew 12% to $4.2M" | Bury in paragraph prose | CalloutBox with bold `+12%` + `$4.2M` |
  1595‚Üí| "latency dropped from 120ms to 75ms" | Long explanatory sentence | CalloutBox or metrics table row |
  1596‚Üí| "Q1‚ÜíQ2‚ÜíQ3: 10%‚Üí25%‚Üí40%" | Inline numbers in text | Chart (matplotlib PNG ‚Üí `Image()`) |
  1597‚Üí| "first...second...third..." steps | Paragraph with ordinal words | Numbered Table or process list |
  1598‚Üí| "compared to last year / vs Q2" | Nested comparisons in prose | Side-by-side comparison table |
  1599‚Üí| "accounted for 60% of total" | Percentage in paragraph | Pie chart or stacked bar chart |
  1600‚Üí
